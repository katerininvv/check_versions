# Update Tracker 🛰

Self-hosted наблюдатель за обновлениями Docker-образов с уведомлениями в Telegram
(на русском языке) и защищённым веб-дашбордом для быстрого доступа к вашим сервисам.

- **Слежение за обновлениями.** Отслеживает продуктивную версию `:latest` образов в Docker Hub.
  Когда выходит новая версия — фиксирует событие.
- **Еженедельный дайджест.** По воскресеньям в 12:00 (MSK по умолчанию) присылает в Telegram
  список обновлений с описанием «что изменилось», переведённым на русский через DeepL.
  Если обновлений нет — так и сообщает.
- **Дашборд с авто-входом.** Плитки-ссылки на ваши сервисы; для ресурсов с HTTP Basic Auth
  открывает их уже авторизованными через встроенный обратный прокси. Учётные данные хранятся
  в зашифрованном виде (AES-256-GCM).
- **Один администратор, HTTPS по IP** с самоподписанным сертификатом, хранилище — SQLite.

## Быстрый старт

Нужны установленные Docker и Docker Compose.

```bash
# 1. Перейдите в каталог проекта
cd update-tracker

# 2. Создайте файл окружения и задайте секреты
cp .env.example .env

# 3. Сгенерируйте мастер-ключ и впишите его в .env (SECRET_KEY=...)
openssl rand -hex 32

# 4. Запустите
docker compose up -d --build

# 5. Посмотрите логи (там будет сгенерированный пароль, если вы не задали ADMIN_PASSWORD)
docker compose logs -f
```

Откройте интерфейс: **https://АДРЕС-СЕРВЕРА:18237**

Браузер предупредит о самоподписанном сертификате — это ожидаемо, добавьте исключение.

## Переменные окружения

Минимум, что нужно задать в `.env`:

| Переменная | Обязательна | Назначение |
|---|---|---|
| `SECRET_KEY` | **да** | Мастер-ключ (подпись сессий + шифрование секретов). `openssl rand -hex 32` |
| `ADMIN_LOGIN` | нет | Логин администратора (по умолчанию `admin`) |
| `ADMIN_PASSWORD` | нет | Пароль. Если пуст — сгенерируется и напечатается в логах при первом запуске |
| `HTTPS_PORT` | нет | Порт наружу (по умолчанию `18237`) |
| `TZ` | нет | Часовой пояс расписания (по умолчанию `Europe/Moscow`) |
| `TELEGRAM_BOT_TOKEN` | нет* | Токен бота. Можно вписать позже в UI → Настройки |
| `TELEGRAM_CHAT_ID` | нет* | ID личного чата. Можно определить кнопкой в UI |
| `DEEPL_API_KEY` | нет* | Ключ DeepL для перевода. Без него описания идут без перевода |
| `GITHUB_TOKEN` | нет | Повышает лимит GitHub API при получении release notes |
| `TLS_SAN` | нет | SAN сертификата. Впишите IP сервера: `IP:203.0.113.10,IP:127.0.0.1,DNS:localhost` |

\* Не обязательны для старта, но без них не будет работать рассылка/перевод. Всё это
настраивается и в веб-интерфейсе на странице **Настройки** уже после запуска.

Секреты, заданные через `.env`, применяются только при **первом** запуске (seed). Дальше
источник истины — настройки в UI (хранятся в БД в зашифрованном виде).

## Настройка Telegram

1. В Telegram напишите **@BotFather**, командой `/newbot` создайте бота и получите токен
   (если бот уже есть — токен у вас уже есть).
2. Впишите токен в **Настройки → Telegram** (или в `TELEGRAM_BOT_TOKEN`).
3. **Напишите своему боту любое сообщение** (`/start`) — это обязательно, иначе бот не сможет
   написать вам первым.
4. На странице Настройки нажмите **«Определить chat_id»** — приложение само подставит ID
   вашего личного чата.
5. Нажмите **«Отправить тестовое сообщение»**, чтобы убедиться, что связь есть.

## Настройка перевода

Перевод описаний на русский — с выбором провайдера в **Настройки → Перевод**:

| Провайдер | Стоимость | Ключ | Примечание |
|---|---|---|---|
| **Google** (по умолчанию) | бесплатно | не нужен | Работает сразу, хорошее качество. Подходит для еженедельного дайджеста |
| **LibreTranslate** | бесплатно | обычно не нужен | Свой контейнер рядом — приватно, ничего наружу не уходит |
| **MyMemory** | бесплатно | не нужен | Есть суточный лимит |
| **DeepL** | платно/лимит | нужен ключ | Высокое качество, но API платный |

По умолчанию включён **Google** — ничего настраивать не нужно, перевод работает из коробки
и бесплатно. Если перевод недоступен (нет сети/провайдера), описания уходят без перевода
(как есть) — рассылка при этом не падает.

### Приватный перевод (LibreTranslate)

Если хотите, чтобы текст не уходил во внешние сервисы, поднимите LibreTranslate своим
контейнером: раскомментируйте блок `libretranslate` в `docker-compose.yml`, затем в
**Настройки → Перевод** выберите провайдер «LibreTranslate» и укажите URL
`http://libretranslate:5000`. Первый запуск скачает языковые модели (несколько сотен МБ).

## Копирование проекта на сервер по SSH

Скачайте архив `update-tracker.zip`, затем с вашего компьютера:

```bash
# 1. Скопировать архив на сервер (замените user и адрес сервера)
scp update-tracker.zip user@203.0.113.10:~/

# Если SSH на нестандартном порту (например 2222), порт указывается ЗАГЛАВНОЙ -P:
#   scp -P 2222 update-tracker.zip user@203.0.113.10:~/

# 2. Зайти на сервер
ssh user@203.0.113.10

# 3. На сервере распаковать и запустить
unzip update-tracker.zip -d update-tracker   # если нет unzip: sudo apt install unzip
cd update-tracker
cp .env.example .env
nano .env                                     # вписать SECRET_KEY и при желании секреты
docker compose up -d --build
```

Альтернатива — `rsync` целой папкой (удобно для последующих обновлений):

```bash
rsync -avz --exclude '.env' ./update-tracker/ user@203.0.113.10:~/update-tracker/
```

## Публикация на GitHub и установка оттуда

В проект уже добавлен `.gitignore`, который не даст закоммитить `.env` с секретами,
базу и сертификаты. Секретов в репозитории не будет — всё чувствительное живёт только
в `.env` (он игнорируется) и в БД на сервере.

### 1. Залить проект в свой репозиторий

Самый простой способ — через GitHub CLI (`gh`), если он установлен:

```bash
cd update-tracker
gh repo create update-tracker --private --source=. --remote=origin --push
```

Либо вручную: создайте пустой репозиторий на github.com, затем:

```bash
cd update-tracker
git init
git add .
git commit -m "Update Tracker"
git branch -M main
git remote add origin git@github.com:ВАШ_ЛОГИН/update-tracker.git
git push -u origin main
```

### 2. Установить на сервере из GitHub

```bash
git clone git@github.com:ВАШ_ЛОГИН/update-tracker.git
cd update-tracker
cp .env.example .env
nano .env                       # вписать SECRET_KEY
docker compose up -d --build
```

Если репозиторий **приватный**, серверу нужен доступ к нему: добавьте на сервере SSH-ключ
(deploy key) в настройках репозитория, либо клонируйте по HTTPS с Personal Access Token.
Публичный репозиторий клонируется без дополнительной настройки.

### Обновление потом

```bash
cd update-tracker
git pull
docker compose up -d --build
```

## Как добавить проект для отслеживания

Пример с `wg-easy`:

1. Перейдите в **Проекты → Добавить проект**.
2. В поле «Ссылка на образ» укажите: `wg-easy/wg-easy:latest`
   (тег можно опустить — по умолчанию `latest`).
3. В «Репозиторий релизов» укажите `https://github.com/wg-easy/wg-easy` — оттуда берутся
   описания изменений (Docker Hub сам по себе их почти не содержит).
4. Сохраните. Приложение сразу сделает базовый замер текущей версии; уведомление придёт,
   когда `:latest` обновится.

Поддерживаются обычные публичные образы Docker Hub: `nginx`, `library/postgres`,
`namespace/repo:tag` и т.п.

## Добавление ресурсов и авто-вход

На странице **Ресурсы** добавляются сервисы, которые появляются плитками на дашборде.
Тип авторизации:

- **Просто ссылка** — плитка открывает URL в новой вкладке. Без авто-входа.
- **HTTP Basic Auth** — самый надёжный авто-вход: приложение проксирует запросы и
  подставляет заголовок авторизации. По клику вы попадаете на сервис уже залогиненным.
- **Форма логина** — best-effort: приложение пытается отправить форму входа с вашими
  данными и проксировать сессию.

### Честно об ограничениях авто-входа

Прозрачный авто-вход на **произвольный** сторонний сервис — задача принципиально хрупкая:

- **Basic Auth** работает надёжно.
- **Форма логина** работает не всегда: многие сервисы используют CSRF-токены, капчу,
  многошаговый вход или нестандартные имена полей — в этих случаях автоматический вход
  может не сработать, и откроется обычная форма логина.
- Прокси читает тело ответа целиком и переписывает редиректы на свой префикс; сложные
  одностраничные приложения с абсолютными путями к ресурсам могут отображаться неполностью.
  Для таких сервисов используйте тип «Просто ссылка».

Прокси не предназначен для крупных файловых загрузок/скачиваний.

## Архитектура

```
app/
  main.py          FastAPI: маршруты, сессии, аутентификация, прокси
  config.py        чтение переменных окружения
  crypto.py        шифрование секретов AES-256-GCM
  database.py      SQLite + доступ к данным
  security.py      хэш паролей (Argon2id) + троттлинг входа
  tracker.py       опрос Docker Hub (digest тега :latest)
  releasenotes.py  получение release notes из GitHub (best-effort)
  translator.py    перевод через DeepL (подключаемый провайдер)
  notifier.py      отправка в Telegram + сборка дайджеста
  jobs.py          опрос проектов + формирование рассылки
  scheduler.py     APScheduler (Вс 12:00; промежуточный опрос — опционально)
  proxy.py         обратный прокси с авто-входом
  templates/       интерфейс (Jinja2)
  static/          стили
```

Стек: Python 3.12 · FastAPI · APScheduler · SQLite · Docker.

## Эксплуатация

```bash
docker compose up -d --build     # запустить/пересобрать
docker compose logs -f           # логи (в т.ч. пароль администратора при 1-м запуске)
docker compose down              # остановить
docker compose restart           # перезапустить
```

**Резервная копия** — это один том `ut_data` (БД + TLS-сертификат). Достаточно сохранить его.
БД лежит внутри тома по пути `/data/app.db`.

**Смена пароля** — в интерфейсе на странице Настройки.

## Безопасность

- Все секреты (токен бота, ключ DeepL, пароли к ресурсам) хранятся в БД в зашифрованном виде
  (AES-256-GCM, ключ выводится из `SECRET_KEY`). Не теряйте `SECRET_KEY` — без него секреты
  не расшифровать.
- Пароль администратора хранится в виде Argon2id-хэша.
- Вход защищён сессионной cookie (HttpOnly, Secure, SameSite) и троттлингом попыток по IP.
- Формы защищены CSRF-токеном.
- Трафик — по HTTPS (самоподписанный сертификат). Сервис рассчитан на одного администратора;
  открывайте порт наружу осознанно (по возможности — через VPN/firewall).
